# for循环中let与var的区别

# for中的var

引用-[for循环中let与var的区别，块级作用域如何产生与迭代中变量i如何记忆上一步的猜想 - 1024搜-程序员专属的搜索引擎 (1024sou.com)](https://www.1024sou.com/article/238866.html#:~:text=按照模糊的理解，当for循环使用let时产生了块级作用域，每次循环块级作用域中的,i 都相互独立，并不像var那样全程共用了一个。)

js中var是不具有块级作用域的，如果不是很了解建议先看看我的上篇文章中变量提升部分

//使用var声明，得到3个3
var a = [];
for (var i = 0; i < 3; i++) {
  a[i] = function () {
    console.log(i);
  };
}
a[0](); //3
a[1](); //3
a[2](); //3

上面代码中，变量i是var命令声明的，在全局范围内都有效，所以全局只有一个变量i。每一次循环，变量i的值都会发生改变，而循环内被赋给数组a的函数内部的console.log(i)，里面的i指向的就是全局的i。也就是说，所有数组a的成员里面的i，指向的都是同一个i，导致运行时输出的是最后一轮的i的值，也就是 3。

for循序的执行顺序是这样的：设置循环变量(var i = 0) ==》循环判断(i<3) ==》满足执行循环体 ==》循环变量自增(i++)

以第一个var声明为例 展开来看

{
  //我是父作用域
  var i = 0;
  if (0 < 3) {
    a[0] = function () {
      //我是子作用域
      console.log(i);
    };
  };
  i++; //为1
  if (1 < 3) {
    a[1] = function () {
      console.log(i);
    };
  };
  i++; //为2
  if (2 < 3) {
    a[2] = function () {
      console.log(i);
    };
  };
  i++; //为3
  // 跳出循环
}
//调用N次指向都是最终的3
a[0](); //3
a[1](); //3
a[2](); //3

# for中的let

相反js中let是具有块级作用域的

//使用let声明，得到0,1,2
var a = [];
for (let i = 0; i < 3; i++) {
  a[i] = function () {
    console.log(i);
  };
}
a[0](); //0
a[1](); //1
a[2](); //2

上面代码中，变量i是let声明的，当前的i只在本轮循环有效，所以每一次循环的i其实都是一个新的变量，所以最后输出的是6。你可能会问，如果每一轮循环的变量i都是重新声明的，那它怎么知道上一轮循环的值，从而计算出本轮循环的值？

这是因为 JavaScript 引擎内部会记住上一轮循环的值，初始化本轮的变量i时，就在上一轮循环的基础上进行计算。

还是上面的例子，我们在let情况下对for循环步骤拆分，代码如下：

var a = []; {
    //我是父作用域
    let i = 0;
    if (i < 3) {
        //这一步模拟底层实现
        let k = i;
        a[k] = function () {
            //我是子作用域
            console.log(k);
        };
    };
    i++; //为1
    if (i < 3) {
        let k = i;
        a[k] = function () {
            console.log(k);
        };
    };
    i++; //为2
    if (i < 3) {
        let k = i;
        a[k] = function () {
            console.log(k);
        };
    };
    i++; //为3
    // 跳出循环
}
a[0](); //0
a[1](); //1
a[2](); //2

基本数据类型与引用数据的概念上提供了一个有趣的例子，代码如下：

var a = []
for (let y = {i: 0}; y.i < 3; y.i++) {
    a[y.i] = function () {
        console.log(y.i);
    };
};
a[0]();//3
a[1]();//3
a[2]();//3

我们还是一样的改成写模拟代码，如下：

        let k = y;
        a[k.i] = function () {
            console.log(k.i);
        };

注意，在模拟代码中为let k = y而非let k = y.i。我们始终使用let声明一个新变量用于保存for循环中的初始变量y，以达到创建块级作用域的目的，即使y是一个对象。

那为什么有了块级作用域，最终结果还是相同呢，这就涉及了深/浅拷贝的问题。由于y属于引用数据类型，let k = y 本质上是保存了变量 y 指向值的引用地址，当循环完毕时，y中的 i 已自增为3。

变量k因为块级作用域的原因虽然也是三个不同的k，但不巧的是大家保存的是同一个引用地址，所以输出都是3了。

我们再次改写代码，说说会输出什么：

var a = []
var b = {i:0};
for (let y = b.i; y < 3; y++) {
    a[y] = function () {
        console.log(y);
    };
};
a[0]();//1
a[1]();//2
a[2]();//3
//其实这个例子中let y = b.i   相当于  let y = 0

# 有关循环迭代的其他函数

# forEach

arr.forEach(function(value, index, array) {}

  1. 遍历数组 value：每个数组元素 index ：元素索引 array：数组本身 function 返回值为 undefined 在forEach 里面 return 不会终止迭代，只会中止本次循环

  2. 如果原数组为简单数据类型不会改变原数组的值

  3. 如果原数组为复杂(引用)数据类型则会改变原数组的值

    小试牛刀，有了上面的经验看看下面的例子输出的结果是什么

    let arr = [1, 2, 3]
arr.forEach(item => {
    item += 3
})
console.log(arr)

    如果不确定可以看看下面这个例子

    let a = 1
let fn = item => {
    // item = a
    item += 3
    return item
}
fn(a)
console.log(a) // 毋庸置疑是  1

根据结论2

let arr = [1, 2, 3]
arr.forEach(item => {
    item += 3
})
console.log(arr)  //[1,2,3]

因为参与运算的其实是arr[0],arr[1],arr[2]，也就是数值参与了运算，最后并没有返回给数组，所以数组本身并的元素值并没有改变

# 浅拷贝与深拷贝

相信上面的例子可以看到了变量值为简单数据类型和复杂数据类型的区别，那不防我们深入看看浅拷贝和深拷贝。

# 浅拷贝

// 原数据
    let obj = {
      uname: 'aaa',
      age: 18,
      like_color: ['red', 'green'],
      info: {
        weight: '90kg',
        height: '150cm',
        three_dimensions: [40, 60, 100],
      },
    }
    // let obj1 = obj   // 一般不叫拷贝，一般叫赋值

    // 浅拷贝
    let obj1 = {}
    for (var k in obj) {
      // console.log(k)
      // console.log(obj[k])
      obj1[k] = obj[k]
    }
    obj1.info.weight = '120kg' // 会影响原数据
    obj1.uname = 'bbb' // 不会影响原数据
    // 新数据
    console.log(obj1)
    // 原数据
    console.log(obj)

堆中obj.like_color属性名中其实存的是一个地址值（引用值）指向后面的数组

console.log(obj1)
 obj1 = {
      uname: 'aaa',
      age: 18,
      like_color: '0x1234',  //['red', 'green']的地址值
      info: '0x2345',//{weight: '90kg',height: '150cm',three_dimensions: [40, 60, 100],}的地址值
    }

js在对内存中开辟一个新对象空间将地址存入 obj1中，但是obj1.like_color中存放的是obj.like_color中的地址值，可以理解为只遍历了原对象的一层最上面一层。

function deepCopy(old) {
            if (Array.isArray(old)) {
                let news = []
                old.forEach((item) => {
                    // console.log(item);
                    if(typeof item === 'object'){
                        news.push(deepCopy(item))
                    }else{
                        news.push(item)
                    }
                })   
                return news        
            } else if(typeof old === 'object'){
                let news = {}
                for (let k in old) {
                    if(typeof old[k] === 'object'){
                        news[k] = deepCopy(old[k])
                    }else{
                        news[k] = old[k]
                    }
                }
                return news   
            } else{
                return old
            }
        }

深拷贝就是在内存中创建一个新的空间，将原对象里面的全部数据拷贝一份，更改原数据并不会影响新的对象中的数据。
